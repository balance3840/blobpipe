# Security Policy

## Reporting a vulnerability

**Please do not open a public GitHub issue for security vulnerabilities.**

Use GitHub's private vulnerability reporting instead — it keeps the report confidential until a fix is released:

1. Go to the [Security tab](https://github.com/balance3840/blobpipe/security/advisories/new) of this repository
2. Click **Report a vulnerability**
3. Fill in the details

Include as much of the following as you can:
- A description of the vulnerability and its potential impact
- Steps to reproduce or a proof-of-concept
- The affected versions

You can expect an acknowledgement within 48 hours.

## Supported versions

| Version | Supported |
|---|---|
| 1.x | ✅ |
| < 1.0 | ❌ |
